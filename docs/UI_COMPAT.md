# Frontend UI Compatibility Baseline

本文档记录 MediaToolbox 前端提取移植阶段的 UI 兼容原则。目标不是复制 [Ben8368/MediaTools-v2](https://github.com/Ben8368/MediaTools-v2) 的复杂实现，而是保留用户熟悉的 NAS 风格桌面布局、视觉节奏和操作路径，并在真实后端接入阶段继续保持体验一致。

## 1. 当前结论

- 当前仓库已包含独立 `frontend/` 工程。
- 技术栈为 Vite / React / TypeScript / Zustand。
- 首屏是 NAS 风格 Web 桌面体验。
- 当前默认接入本仓库 Fastify 后端和真实下载 API。
- Photoshop / AE 等外部创作工具仍未接入。

## 2. 兼容原则

- **布局优先兼容**：左侧状态栏、桌面区、多窗口、启动器、右侧状态面板等空间关系保持稳定。
- **用户路径优先**：下载器、文件管理器、设置、日志等高频入口要延续来源项目的使用记忆。
- **业务逻辑不进前端**：前端只负责参数填写、任务展示、状态呈现和结果入口。
- **真实 API 默认**：组件通过 `@/api` 访问能力，历史 `mockApi/` 仅作为迁移参考或测试夹具。
- **渐进恢复**：先打磨最高频工作流，再恢复 PSD、视频、图片或外部工具联动等复杂能力。

## 3. 当前前端范围

已包含：
- 桌面壁纸和静态图标资产。
- 左侧状态栏。
- 桌面图标。
- 应用启动器。
- 多窗口容器。
- 右侧状态面板。
- 下载器 UI。
- 文件管理器 UI。
- 设置入口。
- 日志入口。

暂缓：
- 真实文件系统读写。
- 真实系统指标。
- Photoshop / AE / PSD 编辑器。
- Electron/Tauri 桌面打包。

## 4. 验收标准

- 用户能沿用来源项目的主要视觉记忆找到功能入口。
- 前端没有把未接入能力伪装成真实后端能力。
- 下载器和文件管理器的错误、空态、加载态可读。
- `npm --prefix frontend run typecheck` 和 `npm --prefix frontend run build` 通过。
- 主观体验通过用户确认后，才能将相关 Feature 标为 `[已完成]`。
